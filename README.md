# ten_bot

Мини-приложение для канала секции по настольному теннису в мессенджере MAX.

## Структура проекта

- `backend/` — серверная логика на Node.js + Express.
- `frontend/` — клиентская часть на React + Vite.
- `README.md` — инструкция по установке и настройке.

## Что реализовано

- Лента новостей с постами, фотографиями, видео, комментариями, реакциями.
- Раздел записи на тренировки с календарем и лимитированными слотами.
- Магазин экипировки с карточками товаров, фотографиями, описанием и кнопкой "Купить".
- Рейтинг игроков с аватаркой, именем, годом рождения, ведущей рукой, играми, победами, поражениями и очками.
- Раздел соревнований с выбором турнира, списком игр, статусом матча и редактированием результата.
- Галерея фото тренировок с сеткой 2x2 и просмотром полноразмерных картинок.
- Разграничение прав: модератор видит кнопку "+" в каждом разделе.

## Установка и запуск

1. Установите зависимости:
   - `cd backend && npm install`
   - `cd frontend && npm install`

2. Настройте `backend/.env` на основе `backend/.env.example`.

3. Убедитесь, что PocketBase запущен и доступен на адресе `POCKETBASE_URL`.

4. Запустите сервер:
   - `cd backend && npm run dev`

5. Запустите клиент:
   - `cd frontend && npm run dev`

6. Откройте приложение в браузере по адресу, который выведет Vite (обычно `http://localhost:5173`).

## Настройка PocketBase

Создайте коллекции в PocketBase с такими полями:

### 1. feed
- `content` — Text
- `comments` — JSON
- `reactions` — JSON
- `videoUrl` — Text
- `attachments` — File (multiple)

### 2. bookings
- `date` — Text
- `startTime` — Text
- `endTime` — Text
- `slots` — Number
- `registrations` — JSON

### 3. shop
- `name` — Text
- `sku` — Text
- `description` — Text
- `sizes` — Text
- `price` — Number
- `images` — File (multiple)

### 4. rating
- `name` — Text
- `birthYear` — Number
- `handed` — Text
- `games` — Number
- `wins` — Number
- `losses` — Number
- `points` — Number
- `avatar` — File

### 5. competitions
- `title` — Text
- `games` — JSON

### 6. gallery
- `caption` — Text
- `photo` — File

> Важно: для полей `attachments`, `images`, `avatar` и `photo` разрешите загрузку файлов.

## Как работает модерация

- Модератор входит через токен, указанный в `backend/.env`.
- Модератор видит кнопку "+" в каждом разделе и может создавать записи, товары, игроков, соревнования, игры и фото.
- Обычные пользователи видят информацию и могут оставлять комментарии, ставить реакции и записываться на тренировки.

## Дальнейшие шаги

1. Создайте администратора PocketBase и заполните `backend/.env`.
2. Соберите фронтенд командой `cd frontend && npm run build`.
3. В продакшн-режиме сервер будет отдавать собранный фронтенд из папки `frontend/dist`.

Если потребуется, я могу дополнительно сделать экспорт схемы PocketBase или добавить загрузку файлов в виде галереи с предпросмотром.
